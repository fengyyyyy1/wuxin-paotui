# 五鑫跑腿总控管理后台

V1.6 总控管理后台已开发完成，并通过人工验收。技术栈为 Vue 3、TypeScript、Vite、Vue Router、
Pinia、Axios 和 Element Plus。

Element Plus 通过`unplugin-auto-import`和`unplugin-vue-components`自动按需导入，
不使用`app.use(ElementPlus)`全量注册。

## 环境

- Node.js 24
- npm 11

## 命令

```bash
npm install
npm run dev
npm run type-check
npm run lint
npm run build
```

- `npm run dev`：启动Vite开发服务器。
- `npm run type-check`：执行Vue和TypeScript类型检查。
- `npm run lint`：执行ESLint检查。
- `npm run build`：执行类型检查并生成生产构建产物。

## 环境变量

开发和生产环境均使用相对 API 前缀：

```text
VITE_API_BASE_URL=/api
```

开发服务器将`/api`代理至`http://localhost:8080`。生产环境由网关或 Nginx
转发，不在前端配置中写死后端主机。

环境文件不得写入真实账号、密码、Token或密钥。

## 本地人工联调

1. 通过 IDEA 启动后端服务。
2. 启动前确认本地环境变量已设置`DB_PASSWORD`。
3. 确认后端监听端口为`8080`。
4. 在`wuxin-admin-web`目录执行：

```bash
npm run dev
```

5. 使用浏览器打开 Vite 输出的本地地址。
6. 使用已绑定`ADMIN`角色的本地测试账号登录。
7. 不要在文档、环境文件或代码中记录真实密码、Token或密钥。

## 商家详情人工联调

1. 完成管理员登录并进入“商家管理”。
2. 在列表中点击任意商家的“查看详情”。
3. 确认页面地址进入`/merchants/{merchantId}`。
4. 确认详情页加载真实`GET /api/admin/merchant/{merchantId}`数据。
5. 核对商家基础信息、审核信息、店铺信息、营业信息和资质信息。
6. 测试“刷新”按钮可以重新加载详情。
7. 测试“返回列表”可以回到商家管理列表。
8. 使用不存在的商家ID访问详情页，确认错误提示和重新加载入口正常。
9. 本阶段不得测试编辑或图片上传。
10. 不要在文档、环境文件或代码中记录真实密码和Token。

## 商家审核与状态操作人工联调

1. 完成管理员登录并进入“商家管理”。
2. 打开待审核商家详情页，确认展示“审核通过”和“审核拒绝”按钮。
3. 点击“审核通过”，填写`auditRemark`，确认请求成功后详情页自动刷新。
4. 打开另一条待审核商家详情页，点击“审核拒绝”，填写`reason`，确认请求成功后详情页自动刷新。
5. 打开审核通过且启用的商家详情页，确认展示“禁用商家”按钮。
6. 点击“禁用商家”，填写`reason`，确认请求成功后详情页自动刷新。
7. 打开审核通过且禁用的商家详情页，确认展示“启用商家”按钮。
8. 点击“启用商家”，确认请求成功后详情页自动刷新；该接口按后端真实Controller定义不发送请求体。
9. 确认审核拒绝状态不展示启用操作。
10. 确认重复提交期间按钮为loading状态且不可重复提交。
11. 不要在文档、环境文件或代码中记录真实密码和Token。
