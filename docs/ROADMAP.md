# 五鑫跑腿固定总体路线

> 本文档是项目唯一、长期有效的总体开发与部署路线。
> `README.md`仅展示路线摘要，`PROJECT_STATUS.md`仅记录当前实际进度和断点。
> 路线发生重大变化时，必须先修改本文档并记录原因。

## 一、项目目标

五鑫跑腿是准备真实运营上线的企业级跑腿平台，不是课程作业或单纯演示项目。

最终系统包括：

1. Spring Boot 后端服务
2. 用户微信小程序
3. 骑手端
4. 商家端
5. 总控管理后台
6. MySQL
7. Redis
8. 对象存储
9. 微信登录
10. 微信支付
11. 消息通知
12. Docker、Nginx、HTTPS 与云服务器部署

```text
├── wuxin-miniapp
├── wuxin-rider-app
├── wuxin-merchant-app
└── docs
```

必做范围：

1. 管理员登录页
2. Token 保存和请求拦截
3. 401、403 统一处理
4. 后台基础布局
5. 商家申请分页
6. 商家详情
7. 商家审核通过
8. 商家审核拒绝
9. 商家启用
10. 商家禁用
11. 状态筛选
12. 分页组件
13. 操作确认弹窗
14. 错误提示
15. 退出登录

1. 数据大屏

验收结果：

1. 管理员登录测试通过

3. 商家申请入驻流程测试通过

3. Docker

6. HTTPS

14. 日志目录
15. 数据备份

上线前必须通过：

1. 所有核心接口测试
2. 用户完整下单流程
3. 骑手完整履约流程
4. 商家完整接单流程
5. 管理员完整审核流程
6. 微信登录测试
7. 微信支付测试
8. 支付回调测试
9. 退款测试
10. 重复请求与幂等测试
11. 权限越权测试
12. SQL 注入和参数校验检查
13. 文件上传安全检查
14. 异常恢复测试
15. 数据备份恢复测试
16. HTTPS 检查
17. 生产日志检查

19. 隐私政策与用户协议

明确禁止：

1. 为赶版本跳过支付回调验签
2. 把数据库密码写进代码或 Git
3. 使用固定 userId 判断权限
4. 在生产环境保留测试接口
5. 在正式数据库保留无意义测试数据
6. 未备份直接执行危险 SQL
7. 未经过人工验收直接宣布上线
8. 小程序不断增加与核心业务无关的功能
9. 为追求页面数量扩大项目范围
10. 未达到上线标准就把版本标为 V2.0

## 八、路线维护规则

1. 新功能开发前必须确认所属版本。
2. 临时讨论不得直接改变版本顺序或扩大当前范围。
3. 非必要功能进入后续候选清单，不插入当前版本。
4. 路线调整必须先修改本文档，并在`CHANGELOG.md`记录原因。
5. V2.0 只有达到真实生产上线标准后才能标记完成。
