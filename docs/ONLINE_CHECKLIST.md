# 五鑫跑腿 V2.0 上线检查清单

最后更新时间：2026-07-22

当前目标：优先达到微信小程序真实上线条件，不继续开发与上线无关的大型模块。

当前上线完成率：62%。

## 一、已完成能力

### 用户端

- 微信登录入口、Token保存、登录态恢复和401处理已完成。
- 用户资料、手机号绑定入口、头像兜底、个人中心和设置页已完成。
- 首页、搜索、跑腿入口、门店列表、门店详情、商品列表、商品详情已完成。
- 购物车、地址管理、结算预览、商品订单创建、跑腿订单创建、支付处理页和订单详情已完成。
- 商家入驻、骑手申请、公益服务、关于五鑫等上线必备入口已存在。

### 商家端

- 登录、入驻、审核状态、工作台、店铺资料、营业状态、分类、商品、订单列表、订单详情、接单、拒单和出餐已完成。
- 商家订单状态可回写后端并被后台查询。

### 骑手端

- 登录、申请、审核状态、工作台、接单大厅、我的配送、订单详情、接单、完成配送、统计和排行榜已完成。
- 骑手订单状态可回写后端并被后台查询。

### 管理端

- Vue3 Admin基础工程、登录、路由守卫、Token、Axios、Element Plus、Pinia和主要运营页面已完成。
- Dashboard、订单、用户、骑手、商家、商品、运营、财务、系统配置、权限和日志中心已完成第一阶段代码。

### 后端与数据库

- Spring Boot 3、Java 21、MyBatis Plus、JWT、统一Result、统一异常和核心业务接口已完成。
- 增量SQL覆盖用户、地址、订单、评价、支付流水、商家、商品、购物车、骑手申请和后台运营配置。
- 订单日志、商家审核日志、管理员操作日志和后台配置能力已存在。

## 二、上线阻塞分级

### P0 必须完成

| 项目 | 当前结果 | 上线影响 |
| --- | --- | --- |
| 生产API地址 | 三个小程序配置仍为`http://localhost:8080` | 小程序正式版无法访问 |
| 微信登录 | 后端真实`code2session`网关存在，但默认关闭；用户端开发Mock仍存在 | 必须切到正式AppID/AppSecret并真机验证 |
| 微信手机号 | 仅Mock手机号网关，真实`getuserphonenumber`未实现 | 手机号绑定无法正式上线 |
| 微信支付 | 只有Mock网关；真实`WECHAT`网关类缺失 | 不能收款 |
| 支付回调 | `/api/payment/wechat/notify`未注册 | 支付成功不能被微信可靠回写 |
| 微信退款 | 未实现退款申请、回调和状态同步 | 拒单、取消已支付订单无法闭环 |
| 文件上传 | 未发现Multipart/OSS/COS上传接口 | 资质和商品图片无法正式运营 |
| 图片存储 | 当前多处要求填写HTTPS图片URL | 缺少统一上传、访问和审核机制 |
| HTTPS与域名 | 未发现Nginx/证书/部署配置 | 不满足微信合法域名要求 |
| 首次部署 | SQL需人工按顺序执行 | 部署易漏脚本 |
| 生产日志 | `application.properties`中Mapper日志为DEBUG | 存在性能和敏感信息风险 |
| 微信审核材料 | 用户协议、隐私政策、服务类目、合法域名和支付配置未形成验收记录 | 审核可能被驳回 |

### P1 建议完成

- Redis、限流、防刷、幂等锁和验证码能力。
- 订阅消息、订单状态通知和客服触达。
- 管理员账号、权限、弱口令、禁用策略和审计复核。
- 生产环境数据库备份、慢SQL检查和索引复核。
- 三端正式AppID、体验版和真机兼容性验收。

### P2 后续优化

- BI、高级统计、多租户、复杂活动、自动派单、实时地图轨迹、财务总账。
- Docker化、CI/CD、监控告警、灰度发布和自动回滚。

## 四、上线倒计时路线

### Day 1：生产配置冻结

- 确认正式小程序AppID、商户号、API证书、APIv3 Key、服务器域名和HTTPS证书。
- 关闭全部Mock默认值，整理生产环境变量。
- 将三端API地址从localhost切换为HTTPS正式域名。

### Day 2：真实微信登录和手机号

- 开启真实`code2session`并真机验证新用户、老用户、Token、过期和401。
- 接入真实`getuserphonenumber`或决定上线首版是否允许不绑定手机号。

### Day 3：真实微信支付

- 实现真实微信JSAPI支付网关。
- 实现`/api/payment/wechat/notify`验签、解密、金额校验和幂等确认。
- 关闭Mock支付并完成真机支付小额回归。

### Day 4：退款与异常订单

- 实现退款申请、退款回调、支付流水状态和订单状态同步。
- 验证商家拒单、用户取消已支付订单和后台人工取消。

### Day 5：上传与图片存储

- 接入腾讯云COS或等价对象存储。
- 补齐商家资质、骑手资质、门店图片、商品图片和头像上传。
- 确认图片HTTPS访问、大小限制、格式限制和失败兜底。

### Day 6：部署与数据库

- 服务器安装Java 21、MySQL 8、Node或静态资源构建环境。
- 按顺序执行全部SQL并回查关键表。
- 配置Nginx反代、HTTPS、日志目录和进程守护。

### Day 7：四端闭环真机验收

- 用户端完成登录、下单、支付、确认收货和评价。
- 商家端完成接单、拒单、出餐。
- 骑手端完成接单、配送和完成。
- 管理端确认Dashboard、订单、用户、商家、骑手、财务和日志同步变化。

### Day 8：微信审核材料

- 配置微信后台合法域名、业务域名、隐私协议、用户协议、类目和服务说明。
- 准备审核账号、测试路径、支付说明和客服联系方式。

### Day 9：提交体验版与预审核

- 上传小程序体验版。
- 邀请内部人员真机走完整流程。
- 修复审核前P0问题。

### Day 10：提交微信审核与上线

- 提交微信审核。
- 审核通过后发布正式版。
- 首日监控登录、支付、订单、退款、错误日志和客服反馈。

## 五、当前结论

项目业务功能代码已经接近完整，但真实上线条件尚未满足。当前不应继续开发RBAC、高级统计或大型后台模块，必须优先完成P0生产依赖。
